# Code of Conduct

## Our Commitment

We as contributors, maintainers, and community members of **Ghost Engine / Termux-Tor-IP-Rotator** are committed to making participation in this project a respectful, professional, and harassment-free experience for everyone.

We want this repository to be a place where people can learn, contribute, report issues, suggest improvements, and collaborate in good faith — regardless of experience level, background, identity, or platform.

This project is built around privacy, research, learning, and open-source collaboration. We expect everyone participating in the community to act with maturity, respect, and responsibility.

---

## Expected Behavior

By participating in this project, you agree to:

* Be respectful and constructive in discussions, pull requests, code reviews, and issue threads
* Give feedback in a helpful, technical, and solution-oriented way
* Assume good intent from other contributors unless clearly proven otherwise
* Respect differences in skill level, knowledge, and experience
* Keep discussions focused on improving the project, documentation, code quality, usability, and security
* Report bugs, vulnerabilities, and issues responsibly
* Follow applicable laws and platform rules while using or contributing to the project
* Use professional and appropriate language in all public interactions related to the repository

---

## Unacceptable Behavior

The following behavior is not acceptable in this project or its community spaces:

* Harassment, threats, intimidation, or personal attacks
* Hate speech, discriminatory language, or abusive remarks of any kind
* Trolling, baiting, flame wars, or intentionally disruptive behavior
* Spamming, advertising, or posting irrelevant/self-promotional content without permission
* Sharing private, personal, or sensitive information about others without consent
* Submitting malicious code, intentionally harmful changes, or deceptive pull requests
* Encouraging or assisting illegal, abusive, or unethical use of the project
* Repeatedly derailing discussions away from the project’s purpose
* Misrepresenting other contributors, maintainers, or the project itself

---

## Scope

This Code of Conduct applies to all project spaces, including but not limited to:

* GitHub issues
* Pull requests
* Discussions
* Documentation contributions
* Commit comments and code reviews
* Any public communication channels directly associated with this project

It also applies when an individual is representing the project or acting in a way that affects the project’s community or reputation.

---

## Responsible Use Reminder

**Ghost Engine is intended for privacy, education, research, testing, and legitimate security-related learning.**

Contributors and users must **not** use this project to support:

* illegal activity
* abuse, harassment, stalking, or evasion of law enforcement
* malicious attacks, fraud, credential theft, or unauthorized access
* harmful or deceptive behavior against individuals, services, or organizations

Discussion or contributions that clearly attempt to push the project toward abusive or unlawful use may be removed at maintainer discretion.

---

## Reporting Issues

If you experience harassment, abusive behavior, security concerns related to community conduct, or serious violations of this Code of Conduct, please report it to the maintainer.

* links/screenshots/context if available

---

## Enforcement

Project maintainers may take any action they believe is appropriate in response to behavior that violates this Code of Conduct, including but not limited to:

* requesting a change in behavior
* removing comments, issues, pull requests, or discussions
* closing or locking discussions
* rejecting contributions
* temporarily limiting participation
* permanently banning users from the project community

Enforcement decisions are made at the discretion of the maintainers and are intended to protect the project, its contributors, and its community.

---

## Contribution Standards

To keep Ghost Engine healthy as an open-source project, contributors are encouraged to:

* open clear, focused issues
* keep pull requests relevant and reasonably scoped
* document meaningful changes
* test changes when possible before submitting them
* avoid introducing breaking behavior without explanation
* discuss major feature changes before implementing them
* respect the project’s goals, scope, and design direction

---

## Maintainer Standards

Maintainers are expected to:

* act professionally and fairly
* review contributions in good faith
* explain major decisions when practical
* enforce this Code of Conduct consistently and responsibly
* prioritize the safety, clarity, and long-term health of the project community
